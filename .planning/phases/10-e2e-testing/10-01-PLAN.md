**DemoPage.ts — Add ExampleSelector locators and helper methods:**

Add these locators to the constructor:
- `exampleSelector`: `page.getByTestId('example-selector')`
- `exampleDropdown`: `page.getByTestId('example-dropdown')`
- `loadExampleButton`: `page.getByTestId('load-example-button')`

Add these helper methods:
- `selectExample(exampleName: string)`: Selects an example from the dropdown by its option text value using `this.exampleDropdown.selectOption({ label: exampleName })`. Note: the option values are the example directory names (e.g., 'bank-account-records').
- `loadExample(exampleName: string)`: Calls selectExample then clicks loadExampleButton and waits for executeButton to become enabled (timeout 10s).
- `getExampleDescription()`: Returns the text content of the description paragraph below the dropdown. Locate it via `page.locator('[data-testid="example-selector"] p')`.

Create `e2e/tests/java-fv-example-loading.spec.ts` with a `test.describe('Java FV Example Loading')` block.

**Common setup:** Each test navigates to `/demo?tool=java-verification` and creates a DemoPage instance. Use `test.beforeEach` for this.

**Tests:**

1. **'ExampleSelector dropdown appears with 3 examples'**
   - Wait for `exampleSelector` to be visible (timeout 10s — needs API to load examples)
   - Assert `exampleDropdown` is visible
   - Count option elements inside the dropdown (excluding the placeholder "-- Load an example --")
   - Assert count equals 3

2. **'can load bank-account-records example'**
   - Wait for exampleSelector to be visible
   - Select 'bank-account-records' from the dropdown
   - Assert description text appears below dropdown (non-empty `p` element inside example-selector)
   - Click loadExampleButton
   - Wait for executeButton to be enabled (timeout 10s)
   - Assert executeButton is enabled (projectId was set)

3. **'can load shape-matching example'**
   - Same flow as test 2 but with 'shape-matching'

4. **'can load payment-types example'**
   - Same flow as test 2 but with 'payment-types'

5. **'selecting an example shows its description'**
   - Wait for exampleSelector
   - Select 'bank-account-records'
   - Assert a `p` element with class `text-muted-foreground` appears inside the example-selector div
   - Assert its text content is not empty
   - Change selection to 'shape-matching'
   - Assert the description text changed

6. **'Load Example button is disabled without selection'**
   - Wait for exampleSelector
   - Assert loadExampleButton is disabled when no example is selected (default "" value)

7. **'dropdown resets after successful load'**
   - Load bank-account-records example (use loadExample helper)
   - After the executeButton becomes enabled, check the dropdown value is reset to "" (the placeholder)

Note: These tests only run in the 'desktop' project since the ExampleSelector UI is identical on mobile. Add `test.skip(({ isMobile }) => isMobile, 'ExampleSelector identical on mobile')` in beforeEach or at describe level. Alternatively, if the test framework allows, use `test.describe.configure({ mode: 'serial' })` if tests share state — but since each test has its own page via beforeEach, parallel is fine.
