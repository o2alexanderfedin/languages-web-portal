  <action>
1. **Add RTK Query endpoints to executionApi.ts:**
   - `getExamples` query: `GET /examples/:toolId` returning `{ examples: ExampleInfo[] }`. Import `ExampleInfo` from `@repo/shared`.
   - `loadExample` mutation: `POST /examples/:toolId/:exampleName` returning `ExampleLoadResponse`. Import `ExampleLoadResponse` from `@repo/shared`.
   - Export `useGetExamplesQuery` and `useLoadExampleMutation` hooks.

2. **Create ExampleSelector.tsx component:**
   - Props: `{ toolId: string | null; onExampleLoaded: (projectId: string) => void; disabled?: boolean }`
   - When `toolId` is non-null, call `useGetExamplesQuery(toolId)` to fetch examples list (skip when toolId is null).
   - Render nothing (return null) when toolId is null or examples list is empty.
   - Render a dropdown (native `<select>` element styled with Tailwind) with:
     - Default option: "-- Load an example --"
     - Each example as `<option>` showing `name` text (the API returns descriptive names like "Bank Account Records")
   - Below the select, show the selected example's `description` as a subtitle in `text-sm text-muted-foreground`.
   - "Load Example" button (use `Button` from `@/components/ui/button`) that calls `useLoadExampleMutation` with `{ toolId, exampleName }`.
   - On successful load, call `onExampleLoaded(response.projectId)` to set the projectId in the parent.
   - Show loading spinner on button while loading. Show error message if load fails.
   - Add `data-testid="example-selector"` on the wrapper div, `data-testid="example-dropdown"` on select, `data-testid="load-example-button"` on button.

   Per locked decisions from CONTEXT.md:
   - Dropdown shows name + one-line subtitle (subtitle shown below dropdown when example selected)
   - Ordered simple to complex (API returns them in order, just render as-is)
   - All files shown immediately when loaded (handled by existing execution flow once projectId is set)

3. **Create ExampleSelector.test.tsx:**
   - Test: renders nothing when toolId is null
   - Test: renders dropdown with examples when toolId provided (mock RTK Query)
   - Test: calls onExampleLoaded with projectId after successful load
   - Test: shows loading state while loading example
   - Use vitest and @testing-library/react following existing test patterns in the codebase.
