  <name>Task 1: Extend production stage with .NET runtime, solver binaries, cs-fv DLL, NuGet cache</name>
  <files>Dockerfile</files>
  <action>
In the existing production stage (which begins with `FROM eclipse-temurin:25-jre-noble AS production`), add the following blocks IN ORDER, BEFORE the existing `RUN mkdir -p /app/uploads` block:

**Block A: Install .NET 8 runtime**
```dockerfile
# Install .NET 8 runtime (Ubuntu Noble built-in apt feed — supports amd64 AND arm64, no Microsoft feed needed)
RUN apt-get update && \
    apt-get install -y dotnet-runtime-8.0 && \
    apt-get clean && \
    rm -rf /var/lib/apt/lists/*
```
CRITICAL: Do NOT add the Microsoft package repository (packages.microsoft.com). On Ubuntu Noble (24.04), `dotnet-runtime-8.0` is in the built-in Ubuntu feed and supports both amd64 and arm64. The Microsoft feed does NOT support ARM64.

**Block B: Copy solver binaries**
```dockerfile
# Copy solver binaries from solver-builder (no curl/unzip tools in production)
COPY --from=solver-builder /usr/local/bin/cvc5 /usr/local/bin/cvc5
COPY --from=solver-builder /usr/local/bin/z3   /usr/local/bin/z3
```

**Block C: Copy cs-fv DLL**
```dockerfile
# Copy cs-fv CLI from dotnet-builder (framework-dependent; requires dotnet-runtime-8.0 above)
COPY --from=dotnet-builder /publish/cs-fv /usr/local/lib/cs-fv
```
This copies the entire publish output directory (not just the .dll) — the directory includes cs-fv.dll and all its runtime dependency files. Invocation: `dotnet /usr/local/lib/cs-fv/cs-fv.dll`

**Block D: Copy NuGet package cache and configure path**
```dockerfile
# Copy pre-seeded NuGet packages from dotnet-builder (enables offline restore at request time)
COPY --from=dotnet-builder /nuget-packages /home/nodejs/.nuget/packages

# NuGet global packages path — must match the directory owned by nodejs below
ENV NUGET_PACKAGES=/home/nodejs/.nuget/packages
```

**Block E: Fix NuGet cache ownership** — modify the existing `RUN mkdir -p /app/uploads` block:
The existing block creates the uploads directory, adds the nodejs group/user, and chowns /app/uploads. Extend it to also chown the NuGet cache:
```dockerfile
RUN mkdir -p /app/uploads && \
    groupadd -g 1001 nodejs && \
    useradd -u 1001 -g nodejs -s /bin/false nodejs && \
    chown nodejs:nodejs /app/uploads && \
    chown -R nodejs:nodejs /home/nodejs
```
The `chown -R nodejs:nodejs /home/nodejs` line must run BEFORE the `USER nodejs` instruction — this ensures the nodejs user can write to its home directory and NuGet cache at runtime.

Do NOT move or remove any existing production stage content. The only changes are:
1. Add Blocks A through D before the existing `RUN mkdir -p /app/uploads` block
2. Modify the existing `RUN mkdir -p /app/uploads` block to add `chown -R nodejs:nodejs /home/nodejs`
