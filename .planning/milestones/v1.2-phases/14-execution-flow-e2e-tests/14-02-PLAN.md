- No serial mode — network-intercepted tests have no Docker dependency and run in parallel

These tests intercept the /api/execute endpoint (or SSE stream endpoint) using page.route(). The existing happy path uses `page.goto('/demo?tool=java-verification')` then loads an example — error tests will upload a file via fileInput and select a tool, then intercept the route before clicking execute.

**Setup pattern for each test:**
```typescript
// Navigate and set up state (file + tool selected so button is enabled)
await exec.goto('tool=java-verification');
// Upload a file via the file input
await page.locator('input[type="file"]').setInputFiles(SAMPLE_ZIP_PATH);
await page.getByTestId('upload-success').waitFor({ state: 'visible', timeout: 10_000 });
// Execute button should now be enabled
await expect(exec.executeButton).toBeEnabled({ timeout: 5_000 });
```

**Test 1: 'server 500 error during execution shows error message'**
- Intercept POST to `/api/execute` (or the SSE URL): `await page.route('**/api/execute**', route => route.fulfill({ status: 500, body: 'Internal Server Error' }))`
- Click exec.executeButton
- Assert a user-visible error message appears: look for any element with text matching /error|failed|something went wrong/i using `page.locator('[data-testid="execution-error"], [role="alert"]').filter({ hasText: /error|failed|something went wrong/i })` with toBeVisible({ timeout: 10_000 })
- Assert execute button re-enables: `await expect(exec.executeButton).toBeEnabled({ timeout: 5_000 })`

**Test 2: 'SSE connection abort mid-stream shows error message'**
- Intercept the SSE stream URL and abort: `await page.route('**/api/execute**', route => route.abort('connectionfailed'))`
- Click exec.executeButton
- Assert user-visible error: same locator pattern as Test 1, toBeVisible({ timeout: 10_000 })
- Assert execute button re-enables after error

**Test 3: 'connection timeout shows error message'**
- Intercept and delay response beyond client timeout using: `await page.route('**/api/execute**', async route => { await new Promise(resolve => setTimeout(resolve, 35_000)); route.abort(); })`
- Set test timeout: `test.setTimeout(60_000)` for this specific test
- Click exec.executeButton
- Assert user-visible error message within 40_000ms: `page.locator('[data-testid="execution-error"], [role="alert"]').filter({ hasText: /error|failed|timed out|timeout|something went wrong/i })`, toBeVisible({ timeout: 40_000 })
- Assert execute button re-enables: `await expect(exec.executeButton).toBeEnabled({ timeout: 5_000 })`

NOTE: If the actual execution endpoint URL differs from `/api/execute`, use `'**/execute**'` as a broader glob pattern to catch SSE stream URLs. The route intercept must be set up BEFORE clicking execute.
