The script must implement exactly this behavior:

```bash
#!/bin/bash
set -euo pipefail

# Wrapper for C# FV CLI - bridges portal interface to cs-fv command
# Interface: hupyy-csharp-verify --input <projectPath>
# Invokes: dotnet cs-fv.dll verify <cs-file> for each .cs file found

DOTNET_BIN="${DOTNET_CMD:-dotnet}"
CS_FV_DLL="${CS_FV_DLL:-/usr/local/lib/cs-fv/cs-fv.dll}"
```

**Argument parsing** — exit 2 if not exactly `--input <path>`:
```bash
if [[ "$#" -ne 2 ]] || [[ "$1" != "--input" ]]; then
  MSG="Usage: hupyy-csharp-verify --input <projectPath>"
  echo "$MSG" >&2
  echo "$MSG" >&1
  exit 2
fi
PROJECT_PATH="$2"
```

**Project path validation** — exit 2 if dir does not exist:
```bash
if [[ ! -d "$PROJECT_PATH" ]]; then
  MSG="Error: Project path does not exist: $PROJECT_PATH"
  echo "$MSG" >&2
  echo "$MSG" >&1
  exit 2
fi
```

**Pre-flight: .csproj check** — exit 2 with TWO lines if no .csproj found (exclude bin/ obj/):
```bash
CSPROJ_COUNT=$(find "$PROJECT_PATH" -name "*.csproj" -not -path "*/bin/*" -not -path "*/obj/*" -type f | wc -l)
if [[ "$CSPROJ_COUNT" -eq 0 ]]; then
  MSG1="Error: No .csproj file found in $PROJECT_PATH"
  MSG2="C# verification requires a .csproj project file"
  echo "$MSG1" >&2; echo "$MSG1" >&1
  echo "$MSG2" >&2; echo "$MSG2" >&1
  exit 2
fi
```

**Collect .cs files scoped to each .csproj directory** — SDK-style .csproj files use implicit globbing (no `<Compile Include=...>` elements), so the correct approach is: locate each .csproj, treat its containing directory as the project root, and collect all .cs files under that root (excluding bin/ obj/). This respects `.csproj` project boundaries per the CONTEXT.md locked decision.

```bash
CS_FILES=()
while IFS= read -r -d '' csproj; do
  PROJ_DIR="$(dirname "$csproj")"
  while IFS= read -r -d '' file; do
    CS_FILES+=("$file")
  done < <(find "$PROJ_DIR" -name "*.cs" \
    -not -path "*/bin/*" \
    -not -path "*/obj/*" \
    -type f -print0)
done < <(find "$PROJECT_PATH" -name "*.csproj" \
  -not -path "*/bin/*" \
  -not -path "*/obj/*" \
  -type f -print0)
```

**Pre-flight: .cs check** — exit 2 with TWO lines if no .cs files:
```bash
if [[ ${#CS_FILES[@]} -eq 0 ]]; then
  MSG1="Error: No .cs files found in $PROJECT_PATH"
  MSG2="C# verification requires at least one .cs source file"
  echo "$MSG1" >&2; echo "$MSG1" >&1
  echo "$MSG2" >&2; echo "$MSG2" >&1
  exit 2
fi
```

**Verification loop** — invoke cs-fv once per file, track highest exit code:
```bash
# Run cs-fv verify for each .cs file
# 2>&1 merges cs-fv stderr into stdout for portal SSE streaming
OVERALL_EXIT=0
for cs_file in "${CS_FILES[@]}"; do
  "$DOTNET_BIN" "$CS_FV_DLL" verify "$cs_file" 2>&1 || OVERALL_EXIT=$?
done

exit $OVERALL_EXIT
```

**IMPORTANT constraints:**
- `set -euo pipefail` at top — fail fast
- Error messages written to BOTH stderr (`>&2`) AND stdout (`>&1`) — CONTEXT.md locked decision
- Exit codes: 2 for wrapper validation failures, 1 (from cs-fv) for verification failures
- `exec` NOT used (can't use exec in a loop) — use `exit $OVERALL_EXIT` instead
- `2>&1` on the dotnet invocation — merges cs-fv stderr into stdout stream
- Do NOT use `cs-fv check` — use `cs-fv verify` (they are different commands)
- Do NOT pass directory to cs-fv verify — pass individual `.cs` file path

After writing the file, make it executable:
